## Native Windows — chạy được gì?

**Cài đặt** bằng PowerShell:
```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

**Hoạt động tốt:**
- CLI cơ bản: `openclaw --version`, `doctor`, `plugins list`
- Gateway chạy native
- Agent local: `openclaw agent --local --agent main ...`
- Daemon dùng Scheduled Task (hoặc fallback Startup folder nếu không có quyền)

**Hạn chế đáng kể:**
- 🔴 **Chưa có companion app** — không có giao diện native Windows kiểu system tray, notifications, v.v.
- 🔴 **Browser automation** trên Windows native có thể gặp vấn đề (thiếu `chrome-sandbox` path, khác biệt về process)
- 🔴 **Một số tool Unix-specific** không chạy (tmux, systemd-based features)
- ⚠️ Daemon dùng Scheduled Task thay vì systemd → supervisor kém mạnh hơn
- ⚠️ `openclaw onboard --non-interactive` cần flag `--skip-health` nếu không có gateway
- ⚠️ PATH có thể cần cấu hình tay sau cài đặt

## WSL2 (khuyên dùng)

- **Full experience**: CLI, Gateway, tooling, browser automation, mọi thứ chạy như trên Linux
- **systemd** hoạt động → `openclaw gateway install` tạo systemd service
- Có thể auto-start WSL khi Windows boot → gateway chạy từ trước khi login
- Có thể expose WSL service ra LAN qua portproxy nếu cần

## Tóm lại

> **Native Windows chạy được CLI + Gateway cơ bản, nhưng WSL2 vẫn là recommended path cho full experience.**

Nếu Thầy muốn chạy native cho gọn nhẹ, dùng được — nhưng nếu muốn dùng hết tooling (browser, exec, skills phức tạp) thì WSL2 ổn định hơn nhiều.
